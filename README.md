# Grade Clicker — Кликер оценок

**Краткое описание**:  
Приложение *Grade Clicker* — интерактивный кликер для набора баллов по курсу «Разработка мобильных приложений». Пользователь кликает по изображению, получая +5 баллов за клик. При достижении пороговых значений (0, 50, 70, 90, 100) меняется отображаемая оценка, фоновое изображение и цвет интерфейса. Приложение демонстрирует работу жизненного цикла Activity и сохранение состояния при изменении конфигурации (например, повороте экрана).

---

## Цель работы

Изучить жизненный цикл Activity в Android, научиться обрабатывать переходы между состояниями, а также реализовать сохранение состояния при изменении конфигурации устройства с помощью `rememberSaveable`. В результате создана интерактивная Compose-приложение, демонстрирующая:
- Callback-методы жизненного цикла (`onCreate`, `onStart`, `onResume`, `onPause`, `onStop`, `onRestart`, `onDestroy`);
- Логирование через класс `Log`;
- Разницу между `remember` и `rememberSaveable`;
- Работу с состоянием в Jetpack Compose.

## Ответы на контрольные вопросы

### 1. Какие три метода вызываются при запуске приложения?
При старте Activity вызываются в порядке:
- `onCreate()`
- `onStart()`
- `onResume()`

### 2. Чем отличается `onPause()` от `onStop()`?
- `onPause()` вызывается, когда Activity теряет фокус (например, открывается диалог или другое приложение частично перекрывает экран). Activity всё ещё видна.
- `onStop()` вызывается, когда Activity становится полностью невидимой (пользователь перешёл в другое приложение или нажал Home). В этом состоянии Activity остаётся в памяти, но не отображается.

### 3. Когда вызывается `onRestart()`?
Метод `onRestart()` вызывается **только при повторном запуске Activity после её остановки** (`onStop()`), например, при возврате из Recents (многозадачности). Он **не вызывается** при первом запуске или после `onPause()` без последующего `onStop()`.

### 4. Что происходит при повороте экрана?
Поворот экрана — это *configuration change*. Android:
- Уничтожает текущую Activity (`onDestroy()`),
- Создаёт новую экземпляр Activity заново (`onCreate()` → `onStart()` → `onResume()`),
- Если состояние не сохранено (например, через `rememberSaveable`), данные теряются.

### 5. В чём разница между `remember` и `rememberSaveable`?
| Функция            | Сохраняет при рекомпозиции | Сохраняет при configuration changes (поворот, язык и др.) |
|--------------------|----------------------------|------------------------------------------------------------|
| `remember`         | ✅                         | ❌                                                         |
| `rememberSaveable` | ✅                         | ✅                                                         |

`rememberSaveable` использует механизм `SavedStateHandle`, чтобы восстанавливать состояние после пересоздания Activity.

### 6. Почему `onCreate()` должен вызывать `super.onCreate()`?
Вызов `super.onCreate(savedInstanceState)` необходим для:
- Инициализации базового класса `Activity`,
- Восстановления состояния из `savedInstanceState` (если есть),
- Корректной работы внутренних механизмов Android (например, Lifecycle, ViewModel, FragmentManager).

Без этого вызова Activity может работать некорректно или завершиться с ошибкой.

### 7. Для чего используется класс `Log`?
Класс `android.util.Log` используется для записи диагностических сообщений в системный лог (Logcat). Позволяет отслеживать:
- Последовательность вызовов lifecycle-методов,
- Ошибки и предупреждения,
- Состояние приложения в реальном времени.

Уровни: `v()` (verbose), `d()` (debug), `i()` (info), `w()` (warning), `e()` (error).

### 8. Какой метод вызывается только один раз за время жизни Activity?
Метод **`onCreate()`** вызывается **ровно один раз** за весь жизненный цикл Activity — при её первоначальном создании. Даже при перезапуске после `onStop()`/`onRestart()` он **не вызывается повторно** — вместо этого вызывается `onRestart()` → `onStart()` → `onResume()`.
